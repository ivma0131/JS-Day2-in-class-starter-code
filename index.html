Part 1 - var vs let/const

Instructions: Open the browser inspector and review the JS code for part 1. Reviewing the output, explain how var scopes differently than let and why we'd use let over var.

Explanation: The scope of var variables is a function-scope meaning that if declared inside a function it can only be accessed within the function and not outside of it (globally). Additionally, it also means that anytime it is declared or redeclared anywhere inside a function, it ignores block-scope and overwrites any previous declaration. So if the var is declared inside the function then later redeclared inside a block statement such as an if statement inside that function, then it overwrites the variable inside the function. Var declaration is useful when you want a variable to be available across an entire function, even in nested blocks.

The scope of a let variable is a block-scope meaning that any let variables declared exist only inside the nearest set of {}. So if the let variable was declared in a function and then later declared inside a block in that function, then the value of the outer let isn't overwritten by the inner block variable it only stays in there. Let is useful when you want temporary variables that won't leak outside their block.

The scope of a const variable is the same block scope rules as let but once it is assigned it cannot be reassinged. Const is useful when you want to ensure a variable never changes reference.
